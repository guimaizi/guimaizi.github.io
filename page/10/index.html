金融危机流程

加息

近代股市上涨下挫、金融危机大多是从控制货币发行权的各国央行利率的变化开始，制定经济政策的政府控制着货币的供给量，在各个经济周期中给予合理的利率，保持市面适当的流动性，维护国家安定团结、健康发展。
上一轮美联储加息干死了房地产次级贷引发全球性的金融危机，在上一次金融危机发生时，美联储为了避免出现1929年的大萧条，刺激经济把利率维持在0.25%，维持7年(09-15)才开始逐渐加息，如今2018年11月利率是2.25%。
可以这样肤浅的理解什么是利率:年化利率0.25%,借一百块一年只需利息0.25块，只要投资产品年回报率高于0.25%，你就可以去找美联储借钱。
加息是为了防止低利率时出现的经济风险，进而加息开始收缩流动性达到释放化解风险的目的，也是为了度过下一次危机留有余地。

杀估值

估值就是价值，任何东西都有价值，米、面、菜、企业这些都是资产，任何资产都会有估值，估值由市场来决定，市场由供给关系决定。
在股市中可以用一些指标来估值，pe(市盈率)、pb(市净率)之类、大多数资产会用pe这个指标来估值，pe=市值÷年利润，所以可以这样理解，你投资了当前价值等于150块的资产，每年产生收益10块，十五年赚回本金，所以pe=15,你透支了十五年的未来进行投资，这十五年也可以理解为预期。
货币供给量和央行利率 放松或紧缩后，货币的流动性就会受到影响，进而影响到资产估值，市面上的资产开始资产重估，流动性放松就pe就放大，流动性收紧pe就缩小。
估值与利率之间的关系:在大米的供给不变的情况下，在低利率时一斤米是一块五毛钱，在高利率时一斤米就可能是一块钱，不是大米变了而是货币的价值变了。 因美联储和各国央行加息，市面流动性锁紧，今年各国股市指数pe就缩小了很多。

杀业绩

市面流动性收紧，投资自然就放缓，资产价格开始重估，大宗商品价格回落，人们对未来感到担忧，消费放缓，这样影响到企业利润。

违约率

流动性放缓和企业利润降低会导致高杠杆的资产破产，从而引发违约率。

失业率

紧缩的流动性和违约率引发大家更加不愿意消费，投资，没有投资就没有就业，失业率就此产生。

释放流动性

失业率后，为了安定团结，低利率时代重新开始，政府重新释放货币，刺激经济。

End

释放流动性还是不能解决就业后，人闲是非多，谁也说不准会怎样，可以参考世界上发生的经济体制改革与政治体制改革，危机驱动重大改革，成功改革开放高速发展几十年，失败的例子很多，无非是战争与犯罪率。

子域名随机收集

取随机数随机组合出(0-9 a-z .)字符串，dns穷举子域名
需要环境python3.5+ mongodb、别的自己pip install *
本来是异步的、但是requests会阻塞就线程了。
超时时间和线程数识网络环境自己设置

# coding: utf-8
'''
Created on 2018年10月15日

@author: guimaizi
'''
import dns.resolver,requests,random,queue,time,threadpool
from pymongo import MongoClient
from bs4 import BeautifulSoup
class start:
    def __init__(self,domain):
        '''
        :domian 子域名收集
        '''
        self.domain=domain
        #超时
        self.timeout=5
        #线程数
        self.thread_num=100
        self.domain_list=[]
        self.result=[]
        #每次任务标记
        self.random=self.generate_random_str(16)
    def generate_random_str(self,randomlength):
        """
        :randomlength 生成一个指定长度的随机字符串
        """
        random_str = ''
        base_str = 'abcdefghigklmnopqrstuvwxyz0123456789.'
        length = len(base_str) - 1
        for i in range(randomlength):
            random_str += base_str[random.randint(0, length)]
        return random_str

    def query(self,domain):
        '''
        dns查询存在域名
        '''
        myResolver = dns.resolver.Resolver()
        myResolver.nameservers = ['8.8.8.8', '8.8.4.4']

        try:
                myAnswers = myResolver.query(domain, "A")
                self.domain_list.append(domain)
        except Exception as e:print(e)
    def get_http(self,domain):
        '''
        http获取:
        {"domain":domain,"url":target_url,"status_code":"%s"%str(r.status_code),"size_html":len(r.text),"title":"%s"%str(self.get_title(r.text)),"mark":self.random,"time":time.strftime('%Y-%m-%d',time.localtime())}
        '''
        try:
            kv={'user_agent':'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/69.0.3497.100 Safari/537.36'}
            target_url='http://'+domain
            r = requests.get(target_url,headers=kv,timeout=self.timeout)
            r.encoding='utf-8'
            data={"domain":domain,"url":target_url,"status_code":"%s"%str(r.status_code),"size_html":len(r.text),"title":"%s"%str(self.get_title(r.text)),"mark":self.random,"time":time.strftime('%Y-%m-%d',time.localtime())}
            print(data)
            self.result.append(data)
        except Exception as e:print(e)
    def get_title(self,html):
        '''
        返回title
        '''
        try:
            soup = BeautifulSoup(html, 'html5lib')
            title = soup.find('title')
            return title
        except:
            return 'not'
    def save_result(self,data_list):
        '''
        #保存数据
        :filename 文件路径及文件名
        :data_list list数据
        '''
        domain=self.domain.replace('.','_')
        f=open('%s.txt'%domain,'a',encoding='utf-8')
        for i in data_list:
            f.write('%s\n'%str(i))
        f.close()
    def threadpool_fun(self,fun,lists,num):
        '''
        thread多线程池

        :fun 函数
        :lists 数据
        :num 线程数
        '''
        q = queue.Queue()
        for i in lists:
            q.put(i)
        lst = [q.get() for i in range(q.qsize())]
        pool = threadpool.ThreadPool(num)
        requestss = threadpool.makeRequests(fun, lst)
        [pool.putRequest(req) for req in requestss]
        pool.wait()
        pool.dismissWorkers(num, do_join=True) 
    def run(self,list_str):
        #去重
        client = MongoClient("localhost", 27017)
        db_target_domian = client.domian
        domain=self.domain.replace('.','_')
        collection =db_target_domian[domain]
        list_domain=[]
        for i in list_str:
            i=i+'.'+self.domain
            if collection.find({"domain": "%s"%i}).count()==0:
                list_domain.append(i)
        print(list_domain)
        #运行
        self.threadpool_fun(self.query, list_domain, self.thread_num)
        time.sleep(self.timeout)
        self.threadpool_fun(self.get_http, self.domain_list, self.thread_num)
        #写入数据
        collection.insert(self.result,manipulate=True)
        time.sleep(self.timeout)
        #self.save_result(self.result)
    def random_str(self):
        list_str=[]
        for i in range(1000):
            list_str.append(self.generate_random_str(random.randint(1,10)))
        #运行
        self.run(list(set(list_str)))
if __name__=="__main__":
    while 1:
        try:
            item=start('qq.com')

如何艺术的刷src

作为一个刷src老油条来说下我的经验，我是13年开始刷src的，蝉联宇宙最厉害src tsrc四年的前五，最好成绩是第三-.-‘’’,积分制上有一年的第一。

13、14年那时候真的是如果稍微有点编程功底基本就可以制霸各大src，那个时代甚至好多厂家没有waf(web防火墙)的概念，而web2.0时代才刚开始一段时间，xss、sql、命令注入、代码执行、越权、弱口令什么鬼的到处都是，所以出了很多从零开始的黑客，历练到今年，基本都已经是独当一面的安全工程师，甚至有的还很出名很牛逼很有钱，16、17年开始行业不一样了，从以前厂商裸奔到waf的普及和安全意识的转变，然后刷src人士就不好过了，一个xss都要先绕过waf，被政策施压、各种黑客的扫描测试、乙方厂家扫描测试，相对当年已经重视和安全了很多。

言归正传，刷src也就是批量挖漏洞，漏洞是如何产生？ 交互处常有漏洞，世界本来很安全，自从有了人，和他人，需要处理和他人各种逻辑关系时，就不安全了。

交互流程:

这个互联网就是基于这个逻辑而存在，就如同你论坛发帖，流程:要把文字编辑好提交(输入)->论坛服务器(处理数据:标识是谁发的、属于哪个版块、哪个时间)->保存进入数据库->调用数据库内数据展示(二次处理，根据标识或者时间顺序展示等)。要至少经过这一个流程你的帖子才能发表在论坛上。

<?php 
/*
php xss 漏洞案例
文件名:1.php
触发:www.xxx.com/1.php?xss=[evilcode]
*/
echo $GET_['XSS'];
?>

以上是xss漏洞的交互逻辑:输入(从表单输入)-处理(前端、后端逻辑处理)-展示，漏洞在于处理输入的数据时没有考虑到数据中可能出现的风险，所以漏洞不在于输入时和展示时，而在于处理时，就如例直接展示输入的数据就造成了xss漏洞。
漏洞是在处理时，而没有输入也不会有处理，这是个有鸡还是先有蛋的问题，这个流程才是风险的所在，那么刷src艺术的关键就在寻找这个流程，就是寻找输入点，寻找到的输入点越多，你找到漏洞的可能性越大，他的流程越是复杂，你能在这一个流程上找到漏洞就越多。

寻找web流程/输入点(web前期情报收集)

如何刷src的第一步就是如何寻找这个流程，web常规流程:
(数据由人输入)->(经过http/https传输至)->(web服务器apache/nginx/iis接收并解码)->(脚本语言处理数据) 这就是一个能接受数据的web服务(网站)，可以通过域名/IP访问。

域名:*.org *.com *.cn *.gov *.[a-Z]
IP:IPV4 0.0.0.0-255.255.255.255 IPV6 0:0:0:0:0:0:0:0 → ::

IP的规则 :大家都有使用过路由器，很多人还操作过，一个家用路由器分配的ip:192.168.0.1-192.168.255.255,而分配的这些ip就叫做 ip段
判断是不是属于某家企业的ip和当前ip属于哪个段，http://ipwhois.cnnic.net.cn/bns/query/Query/ipwhoisQuery.do?txtquery=211.152.128.0&queryOption=ipv4

利用以上域名和ip的规则大量收集web服务流程，是很重要的，能提高寻找到漏洞的概率，不过刷一家大厂的漏洞很难凭借ip来寻找到web服务，因为经过这么多年的攻防对抗，都禁止通过ip直接访问网站来防止恶意解析了，所以高效的方式还是通过域名。

经过我的测试，开源工具Subdomain相比与其他更加高效 数据更多，所以选择Subdomain+自己写的爬虫来做域名收集、url收集、表单收集、资产监控。
https://github.com/subfinder/subfinder

逻辑处理(web安全漏洞测试)

收集到的url： 其实收集到的是一个基于http/s格式可控的输入点。 http/https超文本协议，而所有的web端的输入 返回都遵守这个协议。 处理也是根据这个协议传输的格式进行处理，增\删\改\查,管理数据,是目标的业务场景。
处理中可能出现的漏洞风险: * xss * get_xss * post_xss * dom_xss * sql * 报错注入 * 盲注 * 时间延迟注入 * insert update 注入 * Mysql/Mssql/Oracle/PostgreSQL * 非关系型数据库mongodb * 命令/代码注入 * win/linux命令注入 * php/asp/jsp/.net/nodejs/python_web命令注入 * 越权 * 增/删/改/查/越权登录 * 敏感信息泄露 * 密码泄露 * cookie session泄露 * 个人信息泄露 * 商业/政务/军情信息泄露 * 逻辑漏洞
* 验证码爆破 * 支付修改金额 * xxe * ssrf
* 等

而这些漏洞带来的风险也是是逻辑处理环境而定，银行网站你输入的金额、个人资料也是输入点;核电站等工业设备你输入的参数;摄像头后台、汽车终端、atm等，基于环境暴露出的威胁也不一样。
改变输入内容，把输入内容改为对应以上攻击的恶意代码，就会造成风险危害。
漏洞靶场:http://testphp.vulnweb.com/

什么是艺术？

艺术就是技术的巅峰，如独孤求败、利剑-软剑-重剑-草木竹石皆可为剑-无剑胜有剑，把剑术练至出神入化无剑胜有剑的地步，从此天下在无敌手，这就是艺术。

纯手工测试漏洞、半自动测试漏洞、全自动测试漏洞。而我要打造自己的艺术,资产收集-漏洞测试-生成报告-提交漏洞报告，流程全自动。

我的学习的方法

我干过的行业:机械加工-车铣刨磨、服务员、印刷工、产业线工人、网吧网管、程序员、黑客、目前职业股民 算从事金融行业，每一行都有每一行的技巧，都有需要总结的经验，善于总结经验发现技巧的人就是大家眼中的聪明人，不过我相反，很笨。手笨，嘴笨，脑袋笨，慢慢的就有点害怕自己，还好忍耐力不错，因为没有退路,足够多的好奇心，也足够坚持。

学习一样东西首先要知道学习的目的，比如当黑客能干什么？起初的动力是什么？
可参考国内知名安全论坛:黑客人物专栏,去看看他们的起初的动力、学习经历。基本大多都是好奇、为了耍帅，机缘巧合也比寻常人多么点的坚持，然后就一发不可收拾，从此入坑。

学习是一个打怪升级的过程

第一关:

万事开头难，如何入门是第一道关卡，现在还好，百度一搜一大堆信息资料，但是过于碎片化的信息让人更加看不懂，接下来就想着找个老司机来带，但是老司机的技术底蕴层次不齐，有的还是装出来的老司机，很容易就把新手带到沟里(大a股就通常有这种情况，各种荐股诈骗。),而真正有底蕴的老司机是不愿意带新手的，带新手通常是出力不讨好，随便几句就敷衍过去，这就是第一道关卡，过滤掉很多新手。
如何突破第一道关卡:每个行业都有传说级别的大佬，而大佬其实都有出书或者江湖流传的各种资料信息，如投资界的巴菲特，书那么的多而且真的都是精髓: 如果行业没有出现足够突出的大佬，如信息安全行业: 看销量和口碑买回去，按照书中内容一一来实践，多百度、多谷歌，加几个行业相关的qq群，认识些小伙伴，不懂就去qq群里问:

再多看看行业相关的影视作品培养情怀，金融:《大空头》《华尔街之狼》《商海通牒》《亿万》《成为巴菲特》等，信息安全行业:《天蝎》《我是谁:没有绝对安全的系统》《幽灵》。
多想多试、坚持，总之技术与情怀要并重，有了情怀才有坚持研究技术的信仰，等到略有成就，知道了行业的大概轮廓后，第一关的boss就被你打死了。

第二关:

小有所成 :这一关很好过，可以用时间水到渠成，通过资料阅读和实践积累，试探行业轮廓,发现自己的需求与行业的需求，经无数专业技术实践、把自己变成一个可以成为行业底层一线岗位上的人员，能解决常规技术问题。

第三关：

行业资深砖家 :多思多想、勤奋好学，也是水到渠成，有很多人在第二关末期就改行了，去做营销、管理之类的工作了，坚持到第三关的是真爱，经过第一关和第二关的经验积累、就总结出了自己的一套业内的东西，可能是方案\产品\集成技术\思维逻辑，能迅速高效寻找到问题并解决问题，对非常规、疑难问题也可以有效缓解甚至解决，已成资深行业人士。

第四关

细分子行业专家 :天赋使然，关卡难度系数五颗星，能到第四关就是真正的专家了，每个大点的行业都有细分子行业，细分子行业一般都偏向于研究性，如经济学、有宏观与微观之分、有的侧重国家、有的侧重行业、有的侧重公司、甚至个人对经济的影响;如信息安全、有二进制与web之分，有的侧重进攻、有的是防御、有的linux、有的windows、甚至侧重于编程语言php、nodejs、java，这些都是细分子行业，深入研究过这些细分子行业的，能够说明、解决这个子行业的几乎所有问题的就是细分子行业权威专家，但要突破这一关卡，很难！就如同一个科研人员作出等同于获得了诺贝尔奖的成就，是要解决大问题的，对国家对人类。

第五关

行业权威专家 :凤毛麟角、第四关已经说了能达到第五关都是些什么人，在中国就是中国科学院院士、中国工程院资深院士，作出很大贡献的人，自身专业水平到行业巅峰、甚至多个行业子行业的权威专家、传奇般的存在，思想意识也达到宏观眼光看世界。

每个关卡的江湖地位特性，从行业内会议来说明，第一关和第二关基本都是观众，第三、四关是讲技术的，第四、五关是讲行业未来发展趋势，总结行业发展历史的，一般都是这个套路，新手要是能在入行时找到个第三关这种等级的前辈最为靠谱。

我的8年的信息安全经历，水平可能在第三关初期，在信息安全行业有认识第四关的人士，但是打到第五关的在记忆中是没有的，可能是毕竟信息安全面向大众只有不到30年的历史而且属于计算机科学下的细分行业。一年多的金融行业，金融知识可能在第一关末期，但是金融行业是有第五关传说级别的存在，股神沃伦.巴菲特、70多年的从业经历，千亿美元的净资产,重点是87岁高龄已经说明打怪升级是一生的。

中国改革开放史

改革开放

解决近14亿人民的安定、温饱、富足、团结在人类史上是绝无仅有的，而中国做到了，完成这项壮举的方案叫做“改革开放”，历经三十年。

政治与经济

一旦人们失去信心，一切就结束了。
—巴菲特

政治与经济都建立在信心之上，政治掌握人类的最高实质权力，经济掌握人类最多的物质资源，两者都存于异度空间制约现实空间。

政治学与经济学是意义重大的学科，预测未来与选择未来，可能是一个村、镇、县、市、省、国家的未来，关系千千万万的人，一个国家的发展，一个国家的幸福未来，所以伟大政治家、经济学家都是时代的改革者，改革者是要为时代、历史负责任的，名垂千古与遗臭万年。

经济学家提供未来方案，政治学家选择方案出台政策执行，如我国与美帝当前形势。

我国:供给侧改革,防范风险、治污环保、精准脱贫三大攻坚战为主线，稳增长调结构，高速发展转为高质量发展，平稳度过刘易斯拐点。
对手国美帝:经济复苏、拉动内需、加息美元回流、平衡贸易逆差，维护自己世界主导地位。

现实世界是政治家和经济学家是唯一的实践场所，近代中国史最有成就的伟人，邓小平\蒋经国\赵紫阳\朱镕基,邓公、赵紫阳、朱镕基带领中国奔小康，蒋经国打破政治垄断开启台湾真正的民主自由化，他们在中国近代历史进程中每一个选择都有着重大意义，充满智慧与魄力，都是伟大的改革者，选择成本很高，每一个选择都可能横尸遍野或功垂千古。

  • 邓公引领国人解放思想，促进了改革开放。
  • 赵紫阳打破国有经济体制，执行了中国从计划经济的困境迈向市场化和全球化。
  • 朱镕基执行了在邓时代后最大力度的经济体制改革和政府体制改革，严厉打击既得利益团伙，清理落后低效产能与机构。
  • 蒋经国促进台湾政治民主化，经济上成为亚洲四小龙。

是他们改变了历史，还是历史选择了他们？

经过

邓公促进改革开放的历史环境是在中美建交时，苏联成为世界第一实力霸权，中苏交恶，而中国必须要与美国结盟以此制衡苏联势力，开放经济是不是为了和美国结盟的一种政治需要?

我所认为的毛时代，是历史的选择，而不是单一政治家的选择，在毛时代建国初期，不久就与苏联交恶，也不和美国交朋友，在国际政治上基本没有朋友，国内政治上为了不出现内部危机导致再出现地方军阀割据内战继而进行十年文化大革命，军事上防范外敌进行三线建设、诱敌深入、广积粮深挖洞来抵御北边苏联百万大军、沿海台湾的反扑威胁和周边外部威胁等，可谓是内忧外患，建国不易 维持住胜利果实不易，真不知道是哪位高人提供的文化大革命的方案，高人！毛时代末期，中美建交已经可以说明文革已经结束，中国在国际上有了新的重要政治地位，对于欧美主流社会来看，中国开始有抵御苏联的重要战略地位。

1978年以前邓公还不是中国一把手，而美欧势力支持有左倾开放思想的邓公，给中国提供经济、军事、技术支持，中国经济上开始向欧美学习转变，军事上开始逐渐从苏式武器战法转变为欧美的战略战术，政治上开始转为学习资本主义社会的政治体制，直到1989年64事件，左倾与右倾，不左即右，不右即左，欧美的民主自由化在国内又开始一波新的政治浪潮，内因经济快速发展导致的通胀之类经济问题和青壮年人口红利的到来，外因台湾经国先生民主化了台湾和开放国门后西方文化对国内思想意识造成的冲击，导致出现64这样对改革开放踩了脚急刹车的事件。

我认为邓公在这一次的选择上是最为智慧与魄力的，虽然出现了上千人的流血事件，但是稳定住了局势，换来目前为止中国30年的高速发展、第二大经济体、世界强国！

在改革开放初，如果邓公是决策者，那么赵紫阳就是规划执行者、赵有更为开放的思想意识，在改革实践中为市场化与私有化奠定基础，促使所有权与经营权相分离,打破计划经济垄断，开启了市场竞争化，许多对言论自由和出版自由的限制都有所放松，允许知识分子自由表达。

我认为赵对中国改革开放做出了不可磨灭的贡献，但是他是一个很好的决策执行者，但政治视野高度不足，把国家带到了极左接近不可控的边缘，导致出现了几年的急刹车。

邓公92南巡后，第二轮改革开放重新开始至今，由朱镕基总理领导的国务院在中国特色社会主义的体制下大力度的开始改革，中国金融与货币秩序控制、分税制改革、国有企业改革、国家机关精简以及反腐败等，打破既得利益势力，重新分配社会资源，刺激出了经济新动能，为如今的全球第二大经济体奠定了基础。

在改革开放急刹车后，因邓公的92南巡，朱镕基带领中国重新深踩下油门，继续加大力度改革，大力把很多低效的国企、政府机构、不合理政策清理，其中也经历了改革途中的阵痛，大批国企职工下岗潮，江湖中有称“朱嘴一张，工人下岗”，但也因为改革开放力度之大，下岗职工很快能在沿海城市，7个特区再就业、创业，让中国成为世界工厂，发展至今。

蒋经国先生促进台湾纯西方式政治民主化，结束一党专政，大力发展经济，让台湾变成亚洲四小龙之一，可惜各种竞争力缺失，经济进入横盘期，也是因为促进了民主化为大陆提供了民主化的案例，间接促使了64。

认清自己、顺势而为，就如邓公的话“这是国际的大气候和中国自己的小气候所决定了的，是一定要来的，是不以人们的意志为转移的，只不过是迟早的问题，大小的问题。”，强行抗拒选择错误的后果，每一次都很严重。

当下

维持住当下局势、等待时机更进一步，也就是度过中等收入陷阱、刘易斯拐点成为发达国家。 这就是当局的政策主线，度过这个已经坑了数十个国家的坎。 这个坎的特性:经济增长回落或停滞、民主乱象、贫富分化、腐败多发、过度城市化、社会公共服务短缺、就业困难、社会动荡、信仰缺失、金融体系脆弱等。

结语

中国改革开放史就像一个积弱百年的瘦子(清、中华民国、共和国建国初)，总被比自己高比自己胖的比自己壮的欺负(八国联军、日本、苏联、美国、英国等)，在改革开放中吃大鱼大肉把自己吃成了一个胖子改变了这被欺负的状况，但是胖子是有内部三高隐患的(中等收入陷阱)，现在又开始健身减肥(拥有核心竞争力、高质量发展)。

致敬邓公、致敬先辈，当年邓公的选择让天安门流血换来如今的繁荣昌盛，让我这个90后在和平年代生活了二十多年。
